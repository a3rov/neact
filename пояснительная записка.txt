Хочется ввести небольшую терминологию:
Задание = Таск (в переводе с английского)

Моим проектом является WEBServer+API.
Мой проект размещен на сервере одного из хостингов. И также, ему принадлежит домен neact.ru

Перейдя на страницу, нас встретит главная страница с кратким описанием проекта.
После, если Вы не авторизованы в свой аккаунт, то вам будет доступна кнопка, которая приведет Вас на сайт авторизации/регистрации.
В ином случае Вы сразу попадете на страницу с основным содержимым.

Что после авторизации, что после будучи авторизованным вы попадаете на страницу с заказами (neact.ru/tasks).

Страница с тасками открывает нам возможность создавать Таск или брать в работу чужие. Если вы создаёте задание (Таск),
то Вы переходите на страницу создания Таска, на которой и происходит заполнение формы, с последующим созданием задания.

После, на главной странице создаётся задание. Вы можете перейти в него и ждать, пока придет отклик, либо удалить это задание.
Другой пользователь может оставить отклик на Ваш Таск.

При оставлении отклика Вы получите отклик и сможете принимать этого пользователя, как исполнителя заказа.

После принятия отклика автор Таска может:
 — принять задание, в качестве успешно завершенного
 — отказаться от исполнителя и продолжить поиски.
 У обоих пользователей будет доступен чат для разговоров, а также, после успешного выполнения, автор Таска сможет оставить отзыв,
 который пойдет в профиль исполнителю.

Также существует сам профиль пользователей. На странице каждого профиля видно: аватарку, имя, описание, ссылку на телеграм, отзывы.
А автор профиля может перейти к редактированию.

Страница редактирования включает:
— смена описания, никнейма, ссылки на телеграм
— смена аватарки.


Структура:

api - папка с файлом для взаимодействия по API (neact.ru/api/tasks)

data - папка с файлом salt.ru (для шифровки), файлом базы данных

db - папка с файлом для создания сессии sqlAlchemy и файл для работы с некоторыми запросами для sqlAlchemy

routes - папка с различными blueprint'ами, которые работают при переходах на страницах сайта

static/css/style.css - файл с css настройками
static/img - изображения, включая изображения для профиля

templates - файлы HTML

main.py - основной файл для запуска
